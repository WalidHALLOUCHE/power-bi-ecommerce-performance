# Specification du dashboard

## Objectif metier

Donner une vision claire de la performance e-commerce et identifier les leviers d'amelioration du tunnel de conversion.

## Questions auxquelles le rapport doit repondre

- Quels canaux generent le plus de chiffre d'affaires ?
- Quels produits convertissent le mieux ?
- A quelle etape du tunnel perd-on le plus d'utilisateurs ?
- Quels devices ont le meilleur taux de conversion ?
- Le cout d'acquisition reste-t-il coherent avec la marge ?
- Quels produits ou categories generent le plus de retours ?

## Pages Power BI proposees

### 1. Vue executif

KPI principaux :

- chiffre d'affaires ;
- commandes ;
- taux de conversion ;
- marge brute ;
- cout d'acquisition ;
- taux de retour.

Visuels : cartes KPI, courbe mensuelle, repartition CA par canal.

### 2. Tunnel de conversion

Objectif : visualiser les pertes entre chaque etape.

Etapes :

1. sessions ;
2. vues produit ;
3. ajouts panier ;
4. passages checkout ;
5. commandes.

### 3. Performance produits

Objectif : comparer les categories et produits.

Visuels : top produits par CA, taux de conversion par categorie, marge par produit.

### 4. Acquisition

Objectif : comparer les canaux marketing.

Visuels : CA par canal, cout d'acquisition, ROAS, conversion par canal.

### 5. Rentabilite

Objectif : suivre marge, retours et couts.

Visuels : marge brute par categorie, taux de retour, cout d'acquisition par commande.

## Filtres recommandes

- Periode ;
- pays ;
- canal d'acquisition ;
- device ;
- categorie ;
- produit.
